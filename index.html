Berlin Lions Club Event Network Infrastructure

Project Overview

As Assistant Superintendent of Technology for the Berlin Lions Club, I designed and deployed an enterprise-scale network infrastructure supporting mission-critical operations for one of Connecticut's largest annual fairs, serving 300,000+ attendees over a 10-day run.

Technical Scope

The network spans multiple buildings connected via multimode and single-mode fiber optic backbone. Over 80 managed switches provide wired connectivity for point-of-sale terminals, ticketing systems, vendor operations, and administrative functions. VLAN segmentation isolates traffic between operational systems, public WiFi, and management networks. A centrally managed mesh wireless network provides coverage across the entire fairgrounds. Redundancy measures ensure failover for mission-critical systems like ticketing and registration.

Challenges Solved

The temporary nature of the event requires rapid deployment and teardown each year while maintaining enterprise-grade reliability. Outdoor environmental factors including weather, power fluctuations, and physical security all required engineering around. The network must handle massive concurrent user counts during peak hours while keeping operational systems responsive.

Outcome

The infrastructure achieves 99% uptime during the 10-day event, supporting ticketing, registration, vendor payments, live streaming, and public WiFi. The network has been refined annually, with each iteration improving reliability and capacity.

AirPlay Relay Controller

Project Overview

Built a Dockerized IoT application that turns a Raspberry Pi into an AirPlay audio receiver with automatic amplifier power management. When an Apple device starts streaming audio, the system detects the AirPlay session and triggers a GPIO relay to power on connected amplifiers. When streaming stops, a configurable delay timer powers the amplifiers off automatically.

Technical Highlights

The system uses shairport-sync (v4.3.2) for AirPlay reception with ALSA audio output and SoX resampling for high-quality playback. A metadata pipe monitors playback state changes (pbeg/pend events) and a custom Bash script manages GPIO relay switching with debounce logic. The entire stack runs in a Docker container on BalenaOS, supporting Raspberry Pi 3 and 4 hardware. Configuration is fully environment-variable driven for GPIO pin assignment, active state polarity, turn-off delay, AirPlay device name, volume control, and audio backend selection.

Problem Solved

Eliminates the need to manually power amplifiers on and off in a whole-home or distributed audio setup. The relay controller acts as a bridge between consumer AirPlay devices and professional/consumer amplifier hardware.
